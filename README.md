# Job Alert Bot

Polls company career pages (via Greenhouse/Lever public APIs) and Internshala's
fresher-jobs listings, filters for SDE/frontend/backend/fullstack roles for
freshers in India, and emails you when something new matches.

## What this does NOT do (read this first)

- It does **not** cover "all companies in India." It covers companies you
  explicitly add to `companies.yaml` (via Greenhouse or Lever) plus whatever
  Internshala's fresher-jobs categories surface. See the chat where this was
  built for why "all companies" isn't a solvable problem.
- It does **not** cover Naukri, LinkedIn, or Indeed. Naukri's search API
  requires a signed token generated by obfuscated JavaScript that expires
  per-session — not scriptable without a paid scraping service or a
  Selenium session-hijacking workaround. LinkedIn/Indeed have aggressive
  anti-bot measures (Cloudflare, login walls). If you want these later, the
  realistic paths are a paid scraping API (ScrapingBee, Apify) or manually
  checking those platforms yourself.
- It polls on a schedule (default: every 30 min) — not truly instant.
- Internshala scraping uses HTML parsing, not an API, because Internshala
  doesn't offer one. **This can break** if Internshala changes their page
  layout. If you stop getting Internshala results, that's the likely cause —
  check `internshala_source.py` needs updating.

## Setup (10-15 minutes)

### 1. Create a GitHub repo
Push this folder to a new **private** GitHub repository.

### 2. Set up a Gmail App Password (for sending email)
1. Go to your Google Account → Security → 2-Step Verification (must be enabled).
2. Go to https://myaccount.google.com/apppasswords
3. Generate an app password for "Mail". Copy the 16-character password.

(Not your regular Gmail password — Gmail blocks direct SMTP login with your
normal password for security.)

### 3. Add GitHub repo secrets
In your repo: Settings → Secrets and variables → Actions → New repository secret.

Add:
- `SMTP_USER` — your Gmail address (e.g. yourname@gmail.com)
- `SMTP_PASS` — the 16-character app password from step 2
- `TO_EMAIL` — where you want alerts sent (can be the same Gmail address)

### 4. Add companies
Edit `companies.yaml`. See the comments in that file for how to find a
company's Greenhouse or Lever token (2 minutes per company). Start with
5-10 companies you actually want to work for — quality over quantity, since
each one is manual to add.

### 5. Enable the workflow
Push to GitHub. Go to the "Actions" tab in your repo, and enable workflows if
prompted. The bot will run automatically on the schedule in
`.github/workflows/job-alert.yml` (every 30 min by default).

You can also trigger it manually: Actions tab → "Job Alert Bot" → "Run workflow".

### 6. First run
The first run seeds `seen_jobs.json` with everything currently open — you
won't get an email flood of every existing posting. You'll only be alerted
about postings that appear after that first run.

## Tuning

- **Change poll frequency:** edit the `cron` line in
  `.github/workflows/job-alert.yml`. GitHub Actions free tier: public repos
  get unlimited minutes, private repos get 2,000 min/month — a 30-min cron
  uses ~48 runs/day × a few seconds each, well within that.
- **Adjust keyword filters:** edit `filters.py` — `ROLE_KEYWORDS`,
  `FRESHER_KEYWORDS`, `EXCLUDE_KEYWORDS`, `INDIA_LOCATION_HINTS`.
- **Add more companies:** edit `companies.yaml`.

## Local testing (before pushing to GitHub)

```bash
pip install -r requirements.txt
export SMTP_USER="you@gmail.com"
export SMTP_PASS="your16charapppassword"
export TO_EMAIL="you@gmail.com"
python main.py
```

## Files

- `companies.yaml` — list of companies to poll (Greenhouse/Lever)
- `ats_sources.py` — fetches from Greenhouse/Lever public APIs
- `internshala_source.py` — scrapes Internshala fresher-jobs pages
- `filters.py` — keyword/location matching logic
- `notifier.py` — sends the email digest
- `main.py` — orchestrates everything, handles dedup
- `seen_jobs.json` — auto-generated, tracks what's already been emailed
